## **Title**

Выбор базы данных

## **Status**

Принято

## **Context**

Необходимо выбрать СУБД удовлетворяющую атрибутам качества.

Система хранит:

-  персональные данные учеников;

-  оценки, задания, посещаемость;

Требования:

-  целостность данных;

-  поддержка транзакций;

-  масштабируемость до миллионов записей;

-  поддержка multi-tenant модели;

-  высокая доступность.

## **Decision**

Решено использовать реляционную СУБД PostgreSQL 

-  Данные успеваемости и посещаемости требуют строгой консистентности.

-  Реляционные БД лучше подходят для сложных связей.

-  Реляционные БД лучше подходят для мультитеннантных приложений

## **Consequences**

Готовое бесплатное решение с открытым исходным кодом. Надёжная, гибкая. 

 

## Рассмотренные альтернативы:

### 1\. NoSQL база данных (например MongoDB)

**Плюсы**

-  Хорошая горизонтальная масштабируемость.

**Минусы**

-  Ограниченная поддержка транзакций.

-  Сложность составления аналитических запросов.

-  Повышенные риски неконсистентности данных.

-  Гибкая схема данных может привести к ошибкам ввода

   

### 2\. Реляционная СУБД с горизонтальным масштабированием (например PostgreSQL)

**Плюсы**

-  Поддержка транзакций.

-  Четкая модель данных.

-  Хорошая Масштабируемость.

-  Удобство составления отчетов

**Минусы**

-  Более сложная архитектура.

-  Требует продуманной стратегии шардирования.
